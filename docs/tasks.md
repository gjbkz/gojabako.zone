# Tasks

## 運用ルール

### タスクIDのフォーマット
`T{NNN}`（例: `T001`）
- `T` + 3桁ゼロ埋め連番
- タスク内容を記録するファイルをタスクファイルと呼ぶ
- タスクファイルのファイル名は `{ID}-{slug}.md`
- タスクファイルは `docs/tasks/` に配置する
  - 例: `T001-playwright-mcp.md`
  - スラッグはファイルを見つけやすくするための補助。内容を表す短いkebab-case
- 各タスクの詳細・経過はタスクファイルに記録する

### 新しいタスクを始めるとき
1. このファイルで現在の最大番号を確認し、次の番号でIDを決める
2. このファイルに新しい行を追加（ステータス `[~]`）
3.タスクファイルを作成し、目的・背景・作業内容を記録する
4. ブランチ `{ID}`（例: `T002`）を作成して作業する
5. ソース更新前にタスクファイルに調査結果・実装方針を整理して gjbkz に確認を依頼する

### 既存タスクを再開するとき
1. このファイルでステータスを確認する
2. 該当のタスクファイルを読んで経過・結果を把握してから作業を始める

### タスクが完了したとき
1.タスクファイルに結果・結論を記録する
2. このファイルからそのタスクの行を削除する（タスクファイル自体が永続的な記録になる）

## ステータス凡例
- `[ ]` 未着手
- `[~]` 進行中
