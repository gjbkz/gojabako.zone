# 日本語表現ルール v1

## 1. 目的

表現の統一ルールを定義します。

## 2. 観測結果（既存記事）

- 文体は「です・ます調」が優勢（本文抽出ベースで `です/ます` 341件、`である/だ。` 4件）。
- 句読点は `、。` が主流（`，．` は検出なし）。
- 送り仮名・表記は `できる`、`ください` が優勢（`出来る`、`下さい` は検出なし）。
- 一部にゆれあり。
- `のとき` と `の時` が混在（各4件）。
- `すること` と `する事` が混在（8件 / 1件）。
- 半角数字が中心だが、全角数字が一部混在（例: `src/app/(blog)/2019/gitlab-runner/page.mdx:117`）。
- サービス名の表記ゆれがある（`Next.js` / `nextjs`）。

## 3. ルール（初版）

### 3.1 文体

- 本文は原則「です・ます調」に統一する。
- 見出しは体言止めまたは名詞句を基本とし、文末の `。` は付けない。
- 数式説明や手順の箇条書きでも、地の文は「です・ます調」を維持する。

### 3.2 句読点・記号

- 日本語文の句読点は `、。` を使う。
- 日本語文中の補足は全角かっこ `（ ）` を使う。
- コード、コマンド、識別子、URL、パスはバッククォートで囲み、半角のまま保持する。

### 3.3 かな・漢字の使い分け

- 補助動詞・補助表現はひらがなにする。
- `できる`（`出来る` は使わない）
- `ください`（`下さい` は使わない）
- `こと` は形式名詞として使う場合はひらがなにする。
- `事` は熟語・固有語（例: 事情、事象、記事）のみ許容する。
- `とき` は接続的・機能語的な用法で使う。
- `時` は時刻・時代・名詞的用法（例: 起動時、14時）に限定する。

### 3.4 数字・単位・固有名詞

- 数字は原則半角（`0-9`）を使う。
- 単位は記事内の既存慣例に合わせて詰めて書く（例: `3mm`、`26万円`）。
- プロダクト名・サービス名は公式表記を優先する（例: `Next.js`、`GitHub`）。

## 4. 運用ルール

- 迷った場合は「可読性が高いか」「既存多数派か」「公式表記か」の順で判断する。
- 例外パターンが増えた場合は、本ルールに追記して更新する。
