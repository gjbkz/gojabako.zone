# Tasks

## 運用ルール

### タスクIDのフォーマット
`T{NNN}`（例: `T001`）
- `T` + 3桁ゼロ埋め連番
- タスクファイルのファイル名は `{ID}-{slug}.md`（例: `T001-playwright-mcp.md`）
  - スラッグはファイルを見つけやすくするための補助。内容を表す短いkebab-case
- 各タスクの詳細・経過は `docs/tasks/{ID}-{slug}.md` に記録する

### 新しいタスクを始めるとき
1. このファイルで現在の最大番号を確認し、次の番号でIDを決める
2. このファイルに新しい行を追加（ステータス `[~]`）
3. `docs/tasks/{ID}-{slug}.md` を作成し、目的・背景・作業内容を記録する

### 既存タスクを再開するとき
1. このファイルでステータスを確認する
2. 該当の `docs/tasks/{ID}-{slug}.md` を読んで経過・結果を把握してから作業を始める

### タスクが完了したとき
1. `docs/tasks/{ID}-{slug}.md` に結果・結論を記録する
2. このファイルからそのタスクの行を削除する（タスクファイル自体が永続的な記録になる）

## ステータス凡例
- `[ ]` 未着手
- `[~]` 進行中
